Das Konzept

Sechs Ebenen — vom Sensor bis zum fertigen Produkt

1

Anbinden

Alles was Daten hat, wird verbunden. Hardware (ESP32), REST-APIs, MQTT-Bridges, Software-Agents. Ein Wizard fuehrt durch, Auto-Discovery erkennt Daten automatisch.

2

Verstehen

Daten bekommen Bedeutung. Semantische Typen (Temperatur, GPS, Prozent), Einheiten, Berechnungen. HubEx weiss nicht nur DASS eine Zahl kommt, sondern WAS sie bedeutet.

3

Visualisieren & Steuern

Dashboard Builder mit Drag & Drop. Charts fuer Trends, Gauges fuer Momentanwerte, Toggles und Slider fuer bidirektionale Steuerung. Widget-Typ wird automatisch vorgeschlagen.

4

Automatisieren

Regeln die selbststaendig handeln. Von "Wenn X dann Y" bis zu komplexen Ketten mit Verzweigungen, zeitgesteuerten Aktionen, Email-Versand. 7 Trigger, 7 Actions, AND/OR-Logik.

5

Verbinden & Erweitern

Custom API Builder, Plugin Framework, Export/Import. Eigene Endpoints definieren, eigene Logik einbinden, Templates teilen. HubEx wird zur Plattform.

6

Ausliefern

White-Label Branding, Kiosk-Modus, sichere Embed-Links. Das fertige System als Produkt an Endkunden geben. Der Kunde sieht nur sein Dashboard, nicht die Plattform.

Hardware-Plattform — vom Sensor bis zum fertigen Produkt

Visueller Pin-Konfigurator

Board-Grafik, Pins anklicken, Sensoren zuweisen. Drag & Drop statt Datenblatt.

Code Generator

Aus der visuellen Konfiguration wird kompilierbarer Arduino/ESP-Code. Download als PlatformIO-Projekt.

Bridge & SDK

ESP als WiFi-Bridge fuer Arduino. Python/Node.js SDK fuer Raspberry Pi. Software-Agents fuer Linux/Windows.

Edge Computing

Automationen lokal auf dem ESP. Offline-faehig, Echtzeit-Latenz, Sync bei Reconnect.

Industrie-Protokolle

Modbus RTU/TCP, CAN-Bus, RS485. 30+ Device-Profile fuer Energiezaehler, Wechselrichter, SPS.

Lite Edition

Kompakte Version fuer Raspberry Pi. SQLite, <256MB RAM. Als eingebettetes Produkt nutzbar.
