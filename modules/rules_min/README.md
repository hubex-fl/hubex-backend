# rules_min (Example MIC Module)

This is a minimal, MIC-compliant example module. It does not add new public APIs
or runtime side effects. It exists as a reference for structure and capability gating.

## Capabilities (example)
- `module.rules_min.read`
- `module.rules_min.write`

## Notes
- No endpoints added.
- No worker/side-effect logic.
- Configuration is a local stub only.
